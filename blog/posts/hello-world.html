Hello World — Why I Started This Blog

February 14, 2026 · 3 min read
personal web

Every developer has that moment — that spark when a line of code transforms into something you can see, interact with, and share with the world. For me, that moment came the first time I built something from scratch with nothing but HTML, CSS, and JavaScript. No frameworks. No build tools. Just a text editor and a browser.

This blog is an extension of that same spirit.

Why Write?

I've always believed that the best way to truly understand something is to explain it. Writing forces clarity. When I'm debugging a tricky CSS layout or architecting a new feature, the act of putting my thought process into words helps me spot gaps in my own understanding.

But beyond personal growth, there's something else: I want to share the journey. The web development space can feel overwhelming — new frameworks every week, new paradigms every month. I want to write about the fundamentals that stay constant, the patterns that actually matter, and the small discoveries that make the craft enjoyable.

What to Expect

This blog will cover a mix of topics:

  • Web development deep dives — CSS techniques, JavaScript patterns, performance optimization, and the occasional "TIL" moment
  • Design thinking — how I approach UI/UX decisions, typography choices, and making things feel polished
  • Building in public — documenting the process of creating projects, including the mistakes and dead ends
  • Tools and workflow — my development setup, the tools I rely on, and how I stay productive

The Stack Behind This Blog

If you're curious about the technical side — this entire site (portfolio + blog) is built with vanilla HTML, CSS, and JavaScript. No React. No Next.js. No static site generator. Just clean, handwritten code.

Here's a quick look at how the blog system works:

// Blog posts are defined in a simple JSON file
const response = await fetch('posts.json');
const posts = await response.json();

// Each post is rendered as a card on the listing page
posts.forEach(post => {
  const card = createPostCard(post);
  grid.appendChild(card);
});

Posts are individual HTML files with a shared template. The listing page dynamically loads post metadata from a JSON file. It's simple, fast, and I have complete control over every pixel.

The best tool is the one you understand completely. For me, that's the web platform itself.

What's Next

I'm planning to write about some of the techniques I used to build this portfolio — the cinematic hero reveal, the custom cursor system, the scroll-driven animations — all without any external libraries. If that sounds interesting, stick around.

Thanks for reading. Let's build something great.
